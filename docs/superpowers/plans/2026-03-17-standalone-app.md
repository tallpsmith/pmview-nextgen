# Standalone PMVIEW App Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a standalone Godot application (`pmview-app`) with a retro arcade main menu, letter-materialisation loading transition, and runtime scene building from the projection core.

**Architecture:** New Godot project (`src/pmview-app/`) consuming `pmview-projection-core` and the bridge addon. Three scenes (main menu → loading → host view) connected by a scene manager autoload. `RuntimeSceneBuilder` (C#) instantiates live Godot nodes from `SceneLayout`. GDScript for scene controllers, shaders for visual effects.

**Tech Stack:** Godot 4.6 (.NET), C# (net8.0/Godot.NET.Sdk 4.6.1), GDScript, GLSL shaders, Press Start 2P font

**Spec:** `docs/superpowers/specs/2026-03-17-standalone-app-design.md`

**Note on testing:** Godot is NOT installed in the CI/VM. Godot scene files (.tscn), GDScript (.gd), and shaders (.gdshader) cannot be tested automatically. The user verifies visual/interactive work in their host Godot editor. C# files that use Godot APIs (RuntimeSceneBuilder) can be compiled but not unit tested without the Godot runtime. Pure .NET logic is already tested via pmview-projection-core's xUnit tests.

### Task 3: Create SceneManager autoload

**Files:**
- Create: `src/pmview-app/scripts/SceneManager.gd`

The SceneManager is an autoload singleton that handles scene transitions and
passes data between scenes.

- [ ] **Step 1: Create SceneManager.gd**

```gdscript
extends Node

## Autoload singleton for scene transitions and data passing.
## Registered in project.godot as SceneManager.

## Data passed from main menu to loading scene.
var connection_config: Dictionary = {}

## Built scene graph passed from loading to host view.
var built_scene: Node3D = null

func go_to_loading(config: Dictionary) -> void:
	connection_config = config
	get_tree().change_scene_to_file("res://scenes/loading.tscn")

func go_to_host_view(scene: Node3D) -> void:
	built_scene = scene
	get_tree().change_scene_to_file("res://scenes/host_view.tscn")

func go_to_main_menu() -> void:
	connection_config = {}
	if built_scene:
		built_scene.queue_free()
		built_scene = null
	get_tree().change_scene_to_file("res://scenes/main_menu.tscn")

func quit_app() -> void:
	get_tree().quit()
```

### Task 4: Create RuntimeSceneBuilder

**Files:**
- Create: `src/pmview-app/scripts/RuntimeSceneBuilder.cs`

This is the C# equivalent of TscnWriter — it instantiates live Godot nodes from
a `SceneLayout`. Must use explicit `using` directives (addon convention).

- [ ] **Step 1: Create RuntimeSceneBuilder.cs**

The builder must produce the SAME node hierarchy as TscnWriter. Key structure:

```
HostView (Node3D)
├── MetricPoller (Node, MetricPoller.cs)
│   Endpoint, Hostname set
├── SceneBinder (Node, SceneBinder.cs)
├── Per zone:
│   {ZoneName} (Node3D, metric_group_node.gd)
│   ├── {ZoneName}Bezel (MeshInstance3D, ground_bezel.gd)
│   ├── {ZoneName}Grid (Node3D, metric_grid.gd)
│   │   └── shapes/stacks as children
│   └── Each shape:
│       instance of grounded_bar.tscn or grounded_cylinder.tscn
│       colour, ghost properties set
│       PcpBindable child with PcpBindingResource (if not ghost)
├── TimestampLabel (Label3D + PcpBindable)
└── HostnameLabel (Label3D + PcpBindable)
```

Implementation notes:
- Load packed scenes: `GD.Load<PackedScene>("res://addons/pmview-bridge/building_blocks/grounded_bar.tscn")`
- Load scripts: `GD.Load<Script>("res://addons/pmview-bridge/building_blocks/metric_group_node.gd")`
- Instantiate: `scene.Instantiate<Node3D>()`
- Set properties: `node.Set("colour", new Color(r, g, b, 1))`
- Create resources: `new Resource()` with script set to PcpBindingResource
- Report progress via `IProgress<float>` (0.0 to 1.0)
- Handle zone transforms (position, 90° Y rotation for per-instance zones)
- Handle stacks: create StackGroupNode with mode and member shapes

Refer to `TscnWriter.cs` for the exact property names and values.

The file should use explicit `using` directives:
```csharp
using System;
using System.Collections.Generic;
using Godot;
using PmviewProjectionCore.Models;
```

Method signature:
```csharp
public static Node3D Build(SceneLayout layout, string pmproxyEndpoint,
    IProgress<float>? progress = null)
```

## Chunk 2: Main Menu Scene

### Task 5: Create KITT scanner shader

**Files:**
- Create: `src/pmview-app/shaders/kitt_scanner.gdshader`

- [ ] **Step 1: Write the KITT scanner shader**

A fragment shader that creates a neon gradient sweep across a ColorRect/Panel.
Animated via a `sweep_position` uniform that the button script drives.

```glsl
shader_type canvas_item;

uniform float sweep_position : hint_range(-0.4, 1.4) = -0.4;
uniform float sweep_width : hint_range(0.1, 0.6) = 0.4;
uniform float intensity : hint_range(0.0, 1.0) = 0.0;
uniform vec4 color_left : source_color = vec4(1.0, 0.0, 0.43, 1.0);   // #ff006e
uniform vec4 color_mid : source_color = vec4(0.514, 0.22, 0.925, 1.0); // #8338ec
uniform vec4 color_right : source_color = vec4(0.228, 0.525, 1.0, 1.0); // #3a86ff

void fragment() {
    float dist = abs(UV.x - sweep_position);
    float glow = smoothstep(sweep_width, 0.0, dist);
    float t = clamp((UV.x - sweep_position + sweep_width) / (sweep_width * 2.0), 0.0, 1.0);
    vec4 sweep_color = mix(mix(color_left, color_mid, t), color_right, t);
    COLOR = vec4(sweep_color.rgb, glow * intensity);
}
```

## Chunk 3: Loading Scene and Host View

### Task 7: Create materialise shader

**Files:**
- Create: `src/pmview-app/shaders/materialise.gdshader`

- [ ] **Step 1: Write the materialise shader**

A spatial shader that transitions from wireframe to solid. Controlled by a
`materialise` uniform (0.0 = wireframe, 1.0 = full solid).

```glsl
shader_type spatial;
render_mode unshaded;

uniform float materialise : hint_range(0.0, 1.0) = 0.0;
uniform vec4 wire_color : source_color = vec4(0.228, 0.525, 1.0, 0.2);
uniform vec4 solid_color_top : source_color = vec4(1.0, 0.0, 0.43, 1.0);
uniform vec4 solid_color_bottom : source_color = vec4(0.228, 0.525, 1.0, 1.0);
uniform float glow_pulse : hint_range(0.0, 1.0) = 0.0;

void fragment() {
    // Gradient based on world-space Y (normalized)
    float t = clamp(UV.y, 0.0, 1.0);
    vec4 solid = mix(solid_color_bottom, solid_color_top, t);

    // Wireframe: low alpha, just edges visible
    // Solid: full color with emission
    vec4 base = mix(wire_color, solid, materialise);

    // Glow pulse during transition
    float emission_strength = glow_pulse * 2.0 + materialise * 0.5;

    ALBEDO = base.rgb;
    ALPHA = mix(wire_color.a, 1.0, materialise);
    EMISSION = base.rgb * emission_strength;
}
```

### Task 8: Create loading scene

**Files:**

- Create: `src/pmview-app/scripts/LoadingController.gd`

- [ ] **Step 1: Create LoadingController.gd**

Key responsibilities:
- On `_ready()`: get config from `SceneManager.connection_config`, start pipeline
- Run 6 async phases (mapped to P-M-V-I-E-W letters):
  1. Connect to pmproxy (PcpClientConnection)
  2. Fetch topology (MetricDiscovery.DiscoverAsync)
  3. Discover instances (part of topology)
  4. Select profile (HostProfileProvider.GetProfile)
  5. Compute layout (LayoutCalculator.Calculate)
  6. Build scene (RuntimeSceneBuilder.Build)
- After each phase: materialise that letter (tween `materialise` uniform 0→1,
  flash `glow_pulse`, update status text)
- On error: flash letter red, show error + "PRESS ESC TO RETURN"
- On 100%: zoom-exit animation (scale title up, fade out), then
  `SceneManager.go_to_host_view(built_scene)`

The async pipeline runs in C# (called from GDScript via a C# helper node).
Create a small `LoadingPipeline.cs` that wraps the async calls and emits
signals for each phase completion.

- [ ] **Step 2: Create LoadingPipeline.cs**

C# node attached to the loading scene that runs the async discovery → layout →
build pipeline. Emits signals that LoadingController.gd listens to.

```csharp
// Signals: PhaseCompleted(int phaseIndex, string phaseName)
//          PipelineCompleted(Node3D builtScene)
//          PipelineError(int phaseIndex, string error)
```

Method: `StartPipeline(string endpoint)` — runs all 6 phases, emitting signals.
